Guide 01.
Deploy to GitHub

Drei Befehle. Ein Remote. Kein Bullshit. So pushst du deinen Code in unter 30 Sekunden.

01 init

$ git init

// note: initialisiert dein projekt

02 commit

$ git add .

$ git commit -m "first launch"

// note: staged & committed

03 push

$ git remote add origin <url>

$ git push -u origin main

// note: live on github!

// pro tip: erstelle das Repo auf github.com, bevor du git remote add ausführst. sonst gibt's einen 404.

Guide 02.
Update from your PC

Du hast schon deployed? Dann hier dein Update-Loop. Ändern, committen, pushen — fertig.

01 pull

$ git pull origin main

// note: sync mit github

02 edit

$ code .

// ändere deine files

// note: baue, was du willst

03 commit

$ git add .

$ git commit -m "tweak v2"

// note: stage & commit

04 push

$ git push

// note: live in sekunden

// pro tip: bei git push ohne -u funktioniert's, sobald du einmal gepusht hast. kein remote mehr nötig.
